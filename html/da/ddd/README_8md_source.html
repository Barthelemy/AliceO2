2 AliceO2
3 =======
4 
5 Alice O2 project software. Simulation and reconstraction software for the ALICE experiment at CERN based on ALFA and the FairRoot software.
6 
7 ### Installation of Alfasoft (FairSoft)
8 Before start installing please be sure that your system has all the required libraries (see [FairSoft/DEPENDENCIES](https://github.com/FairRootGroup/FairSoft/blob/master/DEPENDENCIES)) for details.
9 
10 #### Full installation:
11 The full installation will install all packages on [FairSoft](https://github.com/FairRootGroup/FairSoft/tree/dev) + [DDS](https://github.com/FairRootGroup/DDS) + [FairRoot](https://github.com/FairRootGroup/FairRoot/tree/dev) and [AliROOT]()
12 
13 This installation:
14 * Need a fast network connection
15 * will take the __development__ branches for all above packages.
16 * It will install Geant4 Data files
17 * Need about __10 GByte__ of disk space (8.1 for the source and objects files, etc and 2.2 GByte for the installation)
18 
19 ##### Step by step for the full installation
20 1. Install [FairSoft/AlFa](https://github.com/FairRootGroup/FairSoft/tree/dev)
21 
22 we use here "alfa_src" as a directory name, you can change it to what ever you like
23 
24 ```bash
25 git clone https://github.com/FairRootGroup/FairSoft.git alfa_src
26 cd alfa_src
27 ./alfaconfig.sh
28 # 1) gcc (on Linux) 5) Clang (on OSX)
29 # 1) No Debug Info
30 # 2) Internet (install G4 files from internet)
31 # path: ~/AlFa
32 ```
33 
34 To run the tests do:
35 ```bash
36 cd alfa_src/FairRoot/build_for_alfa/
37 make test
38 ```

41 #### Minumum installtion (reconstruction only installation)
42 This installaiton will exclude:
43 1. Simulation engines (Geant3/4)
44 2. Event generators (Pythia6/8)
45 3. VGM, VMC
46 ##### Step by step for the minimum installation
47 Edit the "[recoonly.conf](https://github.com/FairRootGroup/FairSoft/blob/master/recoonly.conf)" file in alfa_src, and set your compiler and installation directory.
48 (if you went to use ROOT 6 switch it on!)
49 ```bash
50 compiler= <your compiler>
51 debug=yes
52 optimize=no
53 geant4_download_install_data_automatic=no
54 geant4_install_data_from_dir=no
55 build_root6=no
56 build_python=no
57 install_sim=no
58 SIMPATH_INSTALL= <ALFA_installation_dir>
59 ```
60 
61 1. Install FairSoft
62 ```bash
63 git clone https://github.com/FairRootGroup/FairSoft.git alfa_src
64 cd alfa_src
65 ./configure.sh recoonly
66 ```
67 2. Install [FairRoot](http://fairroot.gsi.de/?q=node/82)
68 
69 ```bash
70 # Set the shell variable SIMPATH to the installation directory
71 export SIMPATH= ALFA_installation_dir
72 [setenv SIMPATH ALFA_installation_dir]
73 
74 git clone -b dev https://github.com/FairRootGroup/FairRoot.git
75 cd FairRoot
76 mkdir build
77 cd build
78 cmake -DCMAKE_INSTALL_PREFIX="FairRoot_installation_dir" ..
79 make
80 make install
81 ```
82 To run the tests do:
83 
84 ```bash
85 # To run test: make new shell, do not define SIMPATH
86 cd FairRoot/build
87 make test
88 ```

90 ### Install the [AliceO2](https://github.com/AliceO2Group/AliceO2) software
91 
92 If you choosed the minimum installation for ALFA before (in step one above) AliceO2 will not include the simulation and reconstruction packages.
93 
94 Set the variable SIMPATH to your FairSoft/alfasoft installation directory
95 
96 ```bash
97 export SIMPATH=ALFA_installation_dir
98 export FAIRROOTPATH=FairRoot_installation_dir
99 ```
100 
101 ```bash
102 git clone https://github.com/AliceO2Group/AliceO2.git
103 cd AliceO2
104 mkdir build_o2
105 cd build_o2
106 cmake ../
107 # -DBUILD_DOXYGEN=ON ( add this option to cmake to generate the doxygen documentation)
108 make
109 . config.sh [or source config.csh]
110 ```

112 ### Generating the doxygen documentation
113 
114 To automatically generate documentation for the AliceO2 project using Doxygen, set the flag -DBUILD_DOXYGEN=ON when calling cmake; the doxygen documentation will then be generated when calling make. The generated html files can be found in the "doxygen/doc/html" subdirectory of the build directory.
115 
116 Doxygen documantation is also available online [here](http://aliceo2group.github.io/AliceO2/)
117 
118 ### Compiling with custom DDS location
119 
120 To include custom DDS location in the compilation, provide DDS_PATH flag when calling cmake. For example:
121 ```bash
122 cmake -DDDS_PATH="/home/username/DDS/0.11.27.g79f48d4/" ..
123 ```
